# 공인중개사 시험 LLM Wiki

## 개요
이 프로젝트는 Karpathy의 LLM Wiki 패턴을 적용하여 공인중개사 시험의 모든 내용을 체계적으로 정리하는 지식 베이스입니다. LLM이 위키를 점진적으로 구축하고 유지관리합니다.

## 아키텍처

### 3계층 구조

1. **sources/** — 원본 자료 (교재, 기출문제, 법령, 판례 등). 불변의 원천 자료.
2. **wiki/** — LLM이 생성·유지하는 마크다운 파일. 요약, 개념 페이지, 법령 해설, 비교표 등.
3. **CLAUDE.md** (이 파일) — 위키 구조, 규칙, 워크플로우를 정의하는 스키마.

### 디렉토리 구조

```
LLMproject/
├── CLAUDE.md              # 스키마 (이 파일)
├── sources/               # 원본 자료
│   ├── textbooks/         # 교재 내용
│   ├── exams/             # 기출문제
│   ├── laws/              # 법령 원문
│   └── cases/             # 판례
├── wiki/
│   ├── index.md           # 전체 위키 색인
│   ├── log.md             # 작업 로그 (append-only)
│   ├── subjects/          # 시험 과목별 개요
│   │   ├── 부동산학개론.md
│   │   ├── 민법및민사특별법.md
│   │   ├── 공인중개사법령및중개실무.md
│   │   ├── 부동산공법.md
│   │   ├── 부동산공시법.md
│   │   └── 부동산세법.md
│   ├── concepts/          # 핵심 개념 페이지
│   ├── laws/              # 법령 해설 페이지
│   └── practice/          # 기출 분석, 출제 경향
└── tools/                 # 유틸리티 스크립트 (필요 시)
```

## 위키 규칙

### 페이지 형식
모든 위키 페이지는 YAML frontmatter를 포함합니다:
```yaml
---
title: 페이지 제목
category: subjects | concepts | laws | practice
subject: 해당 과목명
parent: 상위 개념 페이지명 (있을 경우)
tags: [관련 태그]
sources: [출처 목록]
created: YYYY-MM-DD
updated: YYYY-MM-DD
importance: high | medium | low  # 출제 빈도 기반
---
```

### 위계구조 원칙
- **세분화 우선**: 하나의 페이지가 여러 독립된 하위 개념을 포함하면 반드시 분리
- **상위 페이지**: 하위 개념의 목록과 요약, 비교표를 제공하는 허브 역할
- **하위 페이지**: 개별 개념의 상세 내용 + 시험 최적화 섹션을 포함
- **parent 필드**: 모든 하위 페이지는 frontmatter에 `parent: 상위페이지명` 기재
- **파일명 규칙**: 하위 개념은 `상위개념_하위개념.md` 형식 (예: `용익물권_지상권.md`)
- **깊이 제한**: 최대 2단계 (과목 → 대분류 → 개별개념)

예시:
```
용익물권.md (허브: 지상권/지역권/전세권 비교표 + 링크)
├── 용익물권_지상권.md (상세 + OX퀴즈)
├── 용익물권_지역권.md (상세 + OX퀴즈)
└── 용익물권_전세권.md (상세 + OX퀴즈)
```

### 교차 참조
- 관련 개념은 `[[개념명]]` 형식으로 링크 (Obsidian 호환)
- 법령 참조: `[[법령명#조항]]` 형식
- 비교가 필요한 개념은 비교표로 정리

### Ingest 워크플로우
1. 새 원본 자료를 `sources/`에 추가
2. LLM이 자료를 읽고 핵심 내용 추출
3. 관련 과목 페이지 업데이트
4. 새 개념/법령 페이지 생성 또는 기존 페이지 보강
5. index.md 업데이트
6. log.md에 작업 기록 추가

### Query 워크플로우
1. 질문에 관련된 위키 페이지 검색
2. 여러 페이지를 종합하여 답변 생성
3. 유용한 답변은 위키에 새 페이지로 저장

### Lint 워크플로우
- 페이지 간 모순 확인
- 고아 페이지 (링크 없는) 식별
- 누락된 개념 탐지
- 오래된 법령 정보 갱신 필요 여부 확인
- 출제 경향 대비 커버리지 점검

## 출제 경향 태그
각 개념의 출제 빈도를 추적합니다:
- `#매년출제` — 거의 매년 출제
- `#자주출제` — 2~3년에 한 번
- `#가끔출제` — 간헐적 출제
- `#신규출제` — 최근 새로 출제된 주제

## 시험 최적화 원칙

### 교재 중심 정리
- 위키의 모든 내용은 **에듀랜드 2026 기본서**를 1차 출처로 한다
- 교재의 구성 순서와 체계를 존중하되, 시험 출제 비중에 따라 강약을 조절한다
- 교재에 없는 내용을 임의로 추가하지 않는다 (법령 원문 보충은 예외)

### 개념 페이지 필수 섹션
모든 개념 페이지는 본문 뒤에 다음 시험공부용 섹션을 포함한다:

#### 1. 핵심 암기 포인트
- 시험에 직접 출제되는 숫자, 기간, 요건을 bullet 형태로 정리
- "~이상/이하/초과/미만" 구분, 기간 기산점 등 정확한 표현 사용

#### 2. OX 퀴즈
- 시험에서 자주 출제되는 지문 형태로 5~10개 OX 문제 작성
- 정답과 간단한 해설 포함
- 교재의 기출 포인트와 함정 지문을 반영

#### 3. 비교 정리 (해당 시)
- 혼동하기 쉬운 유사 개념을 표로 비교
- 예: 전세권 vs 임차권, 허가 vs 신고, 절대적 등록취소 vs 임의적 등록취소

#### 4. 함정 노트
- 시험에서 자주 틀리는 포인트를 "⚠️" 표시로 정리
- "~이다(O)" vs "~이다(X)" 형태로 올바른 내용과 오개념 대비

### 학습 보조 기능
- **비교표**: 유사 개념 비교 (예: 전세권 vs 임차권)
- **요약 카드**: 핵심 암기사항 정리
- **기출 분석**: 연도별 출제 빈도 및 패턴
- **함정 노트**: 자주 틀리는 포인트 정리

### 중요한 점
내용이 틀린점이 있어서는 절대 안되므로, 내용이 생성될 때마다, 자체검증을 실시하고, 틀릴경우 web search를 이용해서 내용을 검증합니다.
